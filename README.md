## Overview
This repository captures my journey through the **30 Day Terraform Challenge**, hosted by the **AWS AI/ML User Group Kenya**, **Meru HashiCorp User Group**, and **EveOps**.  

The goal of this challenge is to **expand my practical Terraform expertise**, strengthen my **cloud infrastructure engineering skills**, and explore professional **Infrastructure as Code (IaC) workflows**. Over thirty days, I am progressively building, testing, and deploying cloud infrastructure while applying modern DevOps practices.

Through this challenge, I am deepening my understanding of **modular architecture, remote state management, CI/CD integration, environment isolation, and infrastructure validation**, developing the capabilities needed to design **scalable, maintainable, and secure cloud environments**.

## Challenge Focus Areas
Daily exercises focus on:

- Terraform CLI workflows and automation  
- Provisioning AWS resources using Terraform modules  
- Modular, reusable infrastructure design  
- Remote state management using **S3 backend with state locking**  
- Environment isolation using **Terraform workspaces**  
- Infrastructure validation and security with **TFLint** and **Checkov**  
- CI/CD pipelines for automated deployments  
- Preparation for the Terraform Associate certification

## Professional Engineering Practices
- **Modular Terraform design** using **Terragrunt**  
- **Version-controlled IaC** leveraging Git and branching strategies  
- **CI/CD pipelines** for automated deployment via **GitHub Actions** or **GitLab CI**  
- **Infrastructure validation and security scanning** with **TFLint** and **Checkov**  
- **Pre-commit hooks** for code consistency and linting  
- Remote state management with **AWS S3 backend** and locking
